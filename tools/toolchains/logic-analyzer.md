# 逻辑分析仪与 PulseView 学习笔记

本笔记记录了基于 24MHz 8通道（8CH）USB 逻辑分析仪与开源软件 PulseView 的完整入门、配置及实践过程，便于日后查阅与持续补充。

## 设备基本参数
- **硬件方案**：基于 Cypress FX2 芯片方案的通用逻辑分析仪
- **采样通道**：8 通道（CH1 - CH8 / 对应软件 D0 - D7） + 1 时钟通道 (CLK) + 1 接地端 (GND)
- **最高采样率**：24MHz
- **配套软件**：PulseView (sigrok 项目前端)

---

## 一、 软件下载与安装流程

### 1. 软件下载
- **官方下载网址**：[sigrok Downloads](https://sigrok.org/wiki/Downloads)
- **步骤**：进入网址后，在 **PulseView** 栏目下，根据操作系统选择对应的版本下载（通常为 Windows 64-bit 安装包）。

### 2. 安装注意事项
- 在安装向导的 **Choose Components**（选择组件）步骤中，建议勾选 **`Example data`**（示例数据）。
  - *说明*：示例数据包含官方提前录制好的各种协议波形文件（`.sr` 格式），可用于在没有连接硬件时进行协议解析练习。

---

## 二、 驱动配置 (Zadig 激活)

由于 Windows 系统默认驱动不兼容，必须使用 Zadig 工具将逻辑分析仪的驱动替换为 `WinUSB`。

### 1. 驱动替换步骤
1. 将逻辑分析仪插入电脑 USB 接口。
2. 打开 PulseView 安装目录下的 **`zadig.exe`**。
3. 在 Zadig 菜单栏中点击 **`Options`** -> 勾选 **`List All Devices`**。
4. 在设备下拉列表中选择对应的逻辑分析仪设备（通常显示为 `USB Device`、`Unknown Device` 或 `Saleae Logic`）。
5. 确认右侧目标驱动选择为 **`WinUSB`**。
6. 点击 **`Replace Driver`** 或 **`Install Driver`**，等待提示安装成功（Success）。

### 2. 注意事项
- **无需常驻后台**：Zadig 仅用于修改底层驱动系统注册，安装成功后即可直接关闭，后续使用时无需重复打开。
- **物理接口变动**：如果更换了电脑的 USB 插口，或电脑无法识别设备时，可能需要重新运行 Zadig 重复上述步骤。

## 三、 硬件接线基础

为保证数据采集的准确性，物理连线需遵循以下规则：

1. **共地连接（核心）**：
   - 必须将逻辑分析仪的 **`GND`** 引脚与被测目标板（如单片机、树莓派等）的 **`GND`** 引脚相连。
   - *原因*：若不共地，采集到的信号波形可能会出现严重的噪声干扰或无法正常识别电平。
2. **信号通道映射**：
   - 逻辑分析仪物理面板上的 **`CH1` 至 `CH8`**，在 PulseView 软件中默认对应为通道 **`D0` 至 `D7`**。

---

## 四、 PulseView 软件基础配置与采集

### 1. 设备关联与连接
1. 打开 PulseView。
2. 点击左上角设备下拉菜单（默认可能显示 `No Device`）。
3. **Step 1 (Choose driver)**：下拉选择 **`fx2lafw (generic driver for FX2 based devices)`**。
4. **Step 2 (Choose interface)**：选择 **`USB`**。
5. 点击 **`Scan for devices using driver above`**。
6. 在 Step 4 列表中选中扫描出的设备（如 `Saleae Logic...`），点击 **`OK`**。

### 2. 采样参数设置
- **采样率（Sample Rate）**：
  - 应设置为被测信号频率的 **4 - 5 倍以上**。
  - 例如：测 9600 波特率串口（约 9.6kHz），可设为 `1MHz` 或 `2MHz`；测试 400kHz I2C，建议设为 `5MHz` 或以上。
  - *带宽限制*：24MHz 采样率在某些电脑的 USB 口或使用分线器时可能会因带宽不足报错，可尝试降至 `16MHz` 或 `12MHz`。
- **采样深度/限制（Limit samples）**：
  - 可设置为单次采集固定数量的样本（如 `1 M samples`）或固定时间（如 `100 ms`）。

## 五、 协议解码与波形实践 (以示例文件为例)

在实际接线测试前，可通过软件内置的示例数据练习基本操作和协议解析。

### 1. 载入示例文件
- 打开文件夹中下载好的示例数据（例如：**`mtk3339_8n1_9600.sr`**，这是一个 GPS 模块的 UART 串口信号）。
- 直接将该 `.sr` 文件拖入 PulseView 窗口中即可完成载入。

### 2. 界面视图操作
- **放大/缩小波形**：将鼠标指针移至波形区，**滚动鼠标滚轮**。
- **平移波形**：按住**鼠标左键**在波形区左右拖拽。

### 3. 应用协议解码器（UART 实例）
1. 点击工具栏上的 **`Show protocol decoders`**（黄绿相间的波形图标）。
2. 在右侧搜索框中输入 **`UART`** 并双击添加。
3. 点击主界面左侧新出现的绿色 **`UART`** 标签进行配置：
   - **`Input RX`**：选择有信号波形的通道。
   - **`Baud rate`**：设置为 **`9600`**（参考文件名中的提示）。
   - **`Data format`**：将默认的 `hex` 更改为 **`ascii`**。
4. 放大波形，即可在波形上方直观地看到解析出的 GPS 串口文本数据（如 `$GPRMC...`）。
