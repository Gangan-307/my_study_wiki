# Wi-Fi 模组温升对性能影响的综合评估报告

## 一、 模组温度区间与发热常识

* **人体感温 (45°C - 50°C)**：此区间皮肤会有明显的“烫手”感，但对芯片而言处于安全的常规工作状态。
* **正常工作表面温度 (50°C - 70°C)**：Wi-Fi 在大功率持续发射（Tx）时，功耗可达 1W 左右，此区间发热属于正常物理现象。
* **工业级极限温度 (85°C)**：大多数标准 Wi-Fi 模组的正常工作环境上限。超过此温度，芯片内部温度（结温）通常会突破 100°C，进入性能衰减区。
* **严重过热温度 (>105°C)**：超出物理设计极限，模组随时可能出现死机、复位或永久性物理损坏。

## 二、 核心器件性能随温度上升的衰减机制

### 1. 晶振（时钟源）—— 导致“连接断开”的主因
* **物理机制**：无源晶振具有 S 型温度特性曲线。温度升高会导致晶体物理形变，引起**频率漂移（温漂）**。
* **临界点**：Wi-Fi 标准（802.11）要求射频频偏必须控制在 **±20 ppm** 以内。
* **过热后果**：当模组温度突破 **85°C**，温漂通常会超过 **±22 ppm**，导致子载波失同步、数据包校验错误（CRC Fail）。表现为**网络频繁断连、搜不到热点**。

### 2. 射频发射电路（PA，功率放大器）—— 导致“信号变弱、网速变慢”
* **物理机制**：高温导致半导体中的**载流子迁移率下降**，PA 晶体管的增益变小。
* **过热后果**：
  * **功率下降**：在 85°C 时，发射功率通常会**衰减 1.5 dB 到 2.0 dB**（实际发射能量减少 30% 以上）。
  * **失真加剧**：信号调制质量（EVM）变差。为维持连接，芯片会自动降低调制阶数，导致**物理网速（吞吐量）大幅回退**。

### 3. 射频接收电路（LNA 与噪声底限）—— 导致“接收能力下降”
* **物理机制**：环境热噪声（$P_N = kTB$）随绝对温度升高而增大；同时，低噪声放大器（LNA）在高温下的自身噪声系数（NF）变大。
* **过热后果**：在 85°C 时，接收灵敏度**恶化 1.8 dB 到 2.5 dB**。原本在弱信号区域（如 -85 dBm）能稳定在线的设备，过热后会直接**掉线或显示无信号**。

## 四、 模组过热在设备端的典型异常现象（总结）

当 Wi-Fi 模组处于过热状态时，在应用层或用户端会观察到以下连锁反应：

1. **“假在线”或“频繁重连”**：设备显示连接了 Wi-Fi，但无法传输数据，或者串口日志中频繁出现 `WIFI_REASON_BEACON_TIMEOUT`（信标超时断连）。
2. **通信速率断崖式下跌**：由于 EVM 变差，设备自动从 11n 高速模式退回到 11b 低速模式，网速极慢。
3. **系统异常重启**：高温导致锁相环（PLL）失锁或外部 Flash 读取出错，触发系统的**看门狗复位（Watchdog Reset）**。
4. **功耗恶性循环**：为了补偿高温带来的功率衰减，部分算法会尝试加大偏置电流，这会导致**功耗进一步增加，发热更加严重**。
