# C 语言 - 数组专题

**数组（Array）**是 C 语言中构造数据类型的重要基础。它是**相同数据类型**的变量在内存中**连续排列**而成的集合。理解数组不仅是掌握高级数据结构（如链表、栈、队列）的基石，也是深入理解指针物理本质的关键。

---

## 一、 核心概念与物理结构

### 1. 连续内存布局
*   数组在内存中占用的是一块**连续的内存空间**。
*   任意元素 `arr[i]` 的物理内存地址可以通过基地址（首地址）直接计算得出：
    $$\text{地址}(\text{arr}[i]) = \text{数组首地址} + i \times \text{sizeof}(\text{元素类型})$$
*   因为是连续存储，所以数组支持 **$O(1)$ 时间复杂度的随机访问**（通过下标直接定位）。

### 2. 数组名退化与函数传参
*   **数组名 `arr`**：在绝大多数表达式中，数组名会被编译器隐式转换为**指向数组首元素地址的指针**（即指向 `&arr[0]` 的指针）。
*   **函数传参限制**：当数组作为参数传递给函数时，它会**退化为指针**。因此，在函数内部无法通过 `sizeof(arr)` 计算出整个数组的长度。
    *   **核心法则**：将数组传给函数时，**必须同时传入数组的长度**。

### 3. 数组大小的计算
*   动态计算数组中元素个数的经典公式：
    $$\text{元素个数} = \frac{\text{sizeof}(\text{整个数组名})}{\text{sizeof}(\text{数组的首元素})}$$
    即：`int len = sizeof(arr) / sizeof(arr[0]);`

## 二、 C 语言数组代码实现

以下程序完整演示了一维数组、二维数组的声明、初始化、内存连续性观察，以及数组作为函数参数的正确姿势。

```c
#include <stdio.h>

// ==========================================
// 1. 数组作为函数参数
// ==========================================
// 注意：此时形参 int arr[] 在底层等价于 int *arr（已经退化为指针）
// 因此，必须显式传递数组长度 size
void print_array(int arr[], int size) {
    printf("在函数内部计算 sizeof(arr): %zu 字节 (退化为指针的大小)\n", sizeof(arr));
    
    printf("数组元素依次为: ");
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]); // 也可以写成 *(arr + i)
    }
    printf("\n");
}

int main() {
    // ==========================================
    // 2. 一维数组的声明、初始化与大小计算
    // ==========================================
    printf("--- 1. 一维数组定义与大小计算 ---\n");
    int scores[5] = {90, 85, 95, 80, 100}; // 声明并初始化一个包含 5 个整型的数组
    
    // 计算数组总字节数和元素个数
    size_t total_bytes = sizeof(scores);
    size_t element_count = sizeof(scores) / sizeof(scores[0]);
    
    printf("scores 数组占用的总字节数: %zu 字节\n", total_bytes);      // 5 * 4 = 20
    printf("scores 数组首元素占用的字节数: %zu 字节\n", sizeof(scores[0])); // 4
    printf("scores 数组中包含的元素个数: %zu 个\n", element_count);     // 5

    // ==========================================
    // 3. 数组作为函数参数传递
    // ==========================================
    printf("\n--- 2. 数组作为参数传递时的退化现象 ---\n");
    // 调用函数，传入数组名（退化为首元素地址）和长度
    print_array(scores, (int)element_count);

    // ==========================================
    // 4. 观察数组内存的物理连续性
    // ==========================================
    printf("\n--- 3. 一维数组内存物理连续性观察 ---\n");
    for (int i = 0; i < 3; i++) {
        // 观察相邻元素的物理地址，每次递增 sizeof(int) = 4 个字节
        printf("scores[%d] 的物理地址: %p\n", i, (void*)&scores[i]);
    }

    // ==========================================
    // 5. 二维数组的声明、初始化与遍历
    // ==========================================
    printf("\n--- 4. 二维数组的定义与行优先遍历 ---\n");
    // 二维数组可以看作是“数组的数组”，C 语言采用行优先（Row-Major）存储
    int matrix[2][3] = {
        {1, 2, 3},
        {4, 5, 6}
    };

    printf("二维数组占用的总字节数: %zu 字节\n", sizeof(matrix)); // 2 * 3 * 4 = 24
    
    // 嵌套循环遍历二维数组
    for (int r = 0; r < 2; r++) {
        for (int c = 0; c < 3; c++) {
            printf("matrix[%d][%d] = %d (地址: %p)\n", 
                   r, c, matrix[r][c], (void*)&matrix[r][c]);
        }
    }

    return 0;
}
```

## 三、 核心知识点与面试重点

| 考察维度 | 知识细节与标准回答 |
| :--- | :--- |
| **越界访问检查** | **C 语言不提供数组越界检查。** 如果访问 `scores[5]`，编译器通常不会报错，但会访问到该数组范围之外的未知内存空间，这会导致**未定义行为（Undefined Behavior）**，如崩溃或数据被篡改。 |
| **数组退化机制** | 为什么不能在函数内部用 `sizeof` 算数组长度？因为数组名作为函数参数传递时，会**退化为指向首元素的指针**，其 `sizeof` 返回的是指针本身的尺寸（4字节或8字节），不再是整个数组的尺寸。 |
| **行优先存储** | C 语言中二维数组是按**行优先**方式在内存中一维展开的。例如上面代码中 `matrix[0][2]`（即数据 `3`）和 `matrix[1][0]`（即数据 `4`）在物理内存中是紧挨着的。 |
| **常量性** | 数组名是一个**指针常量**。不能对数组名进行赋值操作，例如 `int a[5], b[5]; a = b;` 是非法的，因为你不能修改常量指针的值。 |
